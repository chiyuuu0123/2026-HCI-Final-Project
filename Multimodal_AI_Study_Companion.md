# LongMindStudy-龙龙多模态智能学习伙伴软件方案

## 1. 项目名称

**中文名称：** LongMindStudy-龙龙多模态智能学习伙伴

**英文名称：** Multimodal AI Study Companion

## 2. 项目定位

LongMindStudy 是一款面向大学生课程学习的桌面端智能学习软件。项目重点不是做一个以实验研究为主的系统，而是做出一个界面精美、功能完整、交互流畅、可以实际演示和使用的软件原型。

软件围绕“上传资料、理解重点、提问答疑、生成练习、追踪进度、调节状态”这一条完整学习链路设计。用户不需要在阅读器、聊天工具、笔记软件和题库之间来回切换，而是在一个统一的学习工作台里完成课程复习。

## 3. 用户需求

大学生在复习课程时常见的问题包括：

- PPT、PDF 和课堂笔记数量多，整理成本高。
- 很多概念散落在不同页面，难以形成整体知识结构。
- 遇到不懂的内容时，需要快速得到解释和例子。
- 复习时缺少针对课程内容的练习题。
- 做错的题目和薄弱知识点容易遗忘。
- 长时间学习时容易疲劳、焦虑、分心或困惑。
- 鼠标键盘操作频繁，学习和演示过程不够自然。

LongMindStudy 的核心价值是把课程资料转化为可学习、可提问、可练习、可追踪的内容，并通过情绪提醒、音乐推荐和手势操作提升软件体验。

## 4. 产品目标

项目最终希望完成一个可运行、可展示、体验完整的桌面端原型。

核心目标包括：

1. 支持上传 PPT、PDF 等课程资料。
2. 自动提取课程重点、关键词、章节结构和核心概念。
3. 自动生成课程摘要、Qwen + Neo4j 交互式知识图谱和独立思维导图。
4. 支持基于课程内容的 AI 问答。
5. 支持自动生成练习题、答案和解析。
6. 自动记录错题、薄弱知识点和学习进度。
7. 提供个性化复习建议。
8. 通过摄像头识别学习状态，给出疲劳或困惑提醒。
9. 根据学习状态推荐背景音乐或休息建议。
10. 支持基础手势控制，实现翻页、确认、收藏和切题。
11. 打磨一个清晰、美观、适合课堂演示的软件界面。

## 5. 核心用户流程

```text
打开软件
  ↓
上传课程资料
  ↓
系统解析 PPT / PDF
  ↓
生成摘要、关键词、Qwen + Neo4j 知识图谱和思维导图
  ↓
查看思维导图并梳理复习层级
  ↓
浏览资料并向 AI 提问
  ↓
生成练习题并进行自测
  ↓
记录错题和薄弱知识点
  ↓
查看学习报告和复习建议
  ↓
通过音乐、提醒和手势改善学习体验
```

## 6. 软件整体设计

LongMindStudy 由三个核心能力层组成：

- **AI 学习助手层**：负责资料解析、摘要生成、知识图谱、AI 问答、自动出题和学习建议。
- **学习状态辅助层**：负责学习进度、错题追踪、情绪提醒和音乐推荐。
- **自然交互层**：负责摄像头手势识别和无接触式操作。

软件不是单纯的 AI 聊天工具，而是一个以课程学习任务为中心的完整学习工作台。

## 7. 页面与界面规划

### 7.1 首页 / 学习工作台

首页用于展示当前学习状态和常用入口。用户进入软件后可以快速看到最近学习的课程资料、今日学习进度、待复习知识点、错题提醒和推荐操作。

主要内容：

- 最近打开的课程资料
- 今日学习时长
- 当前章节完成度
- 待复习知识点
- 错题数量
- 快速上传入口
- 开始复习按钮
- 当前学习状态和音乐状态

界面风格应清爽、现代、低干扰，突出“马上开始学习”的主操作。

### 7.2 资料阅读页

资料阅读页是核心学习页面。页面可以采用三栏布局：

- 左侧：课程目录和章节列表
- 中间：PPT / PDF 内容预览
- 右侧：AI 助手、摘要、关键词和笔记

用户可以一边查看资料，一边向 AI 提问，也可以把回答保存为笔记。

主要功能：

- PPT / PDF 预览
- 章节目录
- 当前页摘要
- 重点概念高亮
- AI 问答入口
- 笔记保存
- 截图收藏

### 7.3 知识图谱页

知识图谱页用于把课程内容结构化展示。当前方案采用 Qwen 抽取课程概念和关系，并将结果写入本地 Neo4j 图数据库，再在前端渲染为可缩放、可拖拽、可查询的关系网络。它应该是软件中视觉效果最强的页面之一，适合课堂演示。

主要功能：

- 自动生成知识网络，并持久化到本地 Neo4j
- 点击节点查看解释、例子和原文出处
- 用颜色区分已掌握、未掌握和易错知识点
- 按章节、难度或掌握程度筛选
- 支持缩放、拖拽和聚焦节点
- 支持搜索知识点、展开一度邻居和查找两个知识点之间的路径
- 支持导出 Mermaid 文本用于报告或课堂展示

示例结构：

```text
人机交互
├── 可用性
│   ├── 有效性
│   ├── 效率
│   └── 满意度
├── 用户体验
│   ├── 情感体验
│   └── 交互体验
└── 评估方法
    ├── SUS 量表
    ├── 用户访谈
    └── 可用性测试
```

### 7.4 思维导图页

思维导图页用于直接根据课程资料整理更适合复习的层级结构。它不复用知识图谱的复杂关系查询作为主要交互，而是把资料主题、章节标题、关键词和图谱概念重新整理成清晰的树状复习结构，帮助学生快速看清主题层级、复习线索和需要优先理解的内容。

主要功能：

- 优先调用 Qwen 根据课程资料生成独立 mindmap JSON，AI 不可用时可使用本地规则从资料标题、章节标题和关键词兜底生成导图
- 可在知识图谱结果基础上按章节和核心概念补充导图分支，使图谱关系和复习层级形成互补
- 按资料主题生成彩色 SVG 层级分支
- 支持展开和收起分支
- 支持聚焦某个导图主题及其子分支
- 点击节点查看摘要、来源和复习提示
- 支持彩色分支和紧凑层级两种展示方式
- 支持导图缩放，适合课堂展示较大的结构图
- 支持添加子主题、编辑节点和删除节点，便于把自动生成内容整理成个人复习版本
- 支持从导图节点进入自动测验，围绕当前主题生成或筛选练习
- 支持导出 Mermaid mindmap 文本

### 7.5 AI 问答页

AI 问答功能应围绕课程资料，而不是泛泛聊天。系统需要尽量引用资料来源，并用适合学习的方式解释内容。

示例问题：

- 这一章的核心概念是什么？
- 帮我解释这页 PPT。
- 这个知识点可能怎么考？
- 请用简单例子解释认知负荷。
- 这个概念和上一章有什么关系？

主要功能：

- 基于资料回答问题
- 支持连续追问
- 显示相关资料来源
- 支持简洁解释、详细解释、举例解释
- 支持一键保存回答为笔记

### 7.6 自动测验页

自动测验页用于帮助学生从“看懂”进入“会做”。系统可以根据当前课程资料、章节、知识点和历史答题情况生成自适应题组。

支持题型：

- 单选题
- 多选题
- 判断题
- 简答题
- 概念匹配题

主要功能：

- 自适应练习、快速诊断、薄弱巩固、今日复习和考前冲刺
- 按章节、知识点、难度和资料来源组织题目
- 自动批改客观题
- 使用 Qwen 对简答题进行语义评分，Qwen 不可用时回退关键词评分
- 展示答案、解析、来源片段和缺失要点
- 错题自动加入错题本并生成今日复习队列
- 根据答题情况推荐复习内容

### 7.7 错题本与学习报告页

学习报告页帮助用户知道自己学得怎么样、接下来该复习什么。

主要功能：

- 学习时长统计
- 章节完成度
- 测验正确率
- 错题本管理
- 薄弱知识点识别
- 学习趋势图
- 个性化复习建议

页面应以可视化卡片和图表为主，让用户快速扫一眼就能知道学习状态。

### 7.7 情绪状态与音乐页

系统通过摄像头识别学习者状态，例如专注、疲劳、焦虑、困惑等，并给出温和提醒。

示例反馈：

> 检测到你可能已经有些疲劳，建议休息 3 分钟，或切换到轻量复习模式。

音乐推荐规则示例：

| 学习状态 | 推荐音乐类型 | 系统策略 |
| --- | --- | --- |
| 专注 | 轻音乐、白噪音 | 保持低干扰播放 |
| 疲劳 | 舒缓但有节奏的音乐 | 提醒休息或降低学习强度 |
| 焦虑 | 放松音乐、自然声 | 降低刺激感 |
| 开心 | 轻快音乐 | 保持积极学习状态 |
| 困惑 | 降低音量或暂停 | 建议查看知识图谱或向 AI 提问 |

### 7.8 手势交互

手势交互用于提升软件的自然感和演示效果，尤其适合阅读资料、做题和课堂展示。

| 手势 | 功能 |
| --- | --- |
| 张手 | 下一页 / 展开知识点 |
| 握拳 | 暂停学习 / 暂停音乐 |
| 点赞 | 确认答案 |
| 比 V | 截图 / 收藏当前内容 |
| 左右挥手 | 上一题 / 下一题 |

手势识别应提供明确的状态提示，避免用户不知道系统是否识别成功。

## 9. 技术方案

### 9.1 前端技术

- Electron
- 原生 HTML / CSS / JavaScript
- SVG 边层 + HTML 节点层图谱渲染
- Lucide 图标

### 9.2 后端技术

- Electron 主进程 IPC
- Qwen / DashScope OpenAI-compatible API
- Neo4j Community 本地图数据库
- Neo4j Desktop 本地 DBMS 管理，并可使用 Desktop 自带 Neo4j/Java 运行时启动项目专用本地库
- 本地 RAG 检索与文档归一化
- PDF 文本提取与 OCR

### 9.3 多模态识别

- MediaPipe
- 摄像头实时预览、状态采样和手势识别

### 9.4 音乐推荐

- 本地音乐规则和固定语音资源
- 基于规则的推荐算法
- 用户反馈优化机制

## 10. 界面设计原则

为了让软件看起来像一个真正可用的产品，界面设计应遵循以下原则：

1. **清晰优先**：核心学习内容放在视觉中心，辅助信息放在侧边栏或状态栏。
2. **低干扰**：避免过多装饰，减少学习时的视觉噪音。
3. **状态可见**：资料解析、AI 生成、摄像头识别、音乐播放和手势识别都要有明确反馈。
4. **操作顺手**：上传、提问、生成题目、保存笔记和查看报告应尽量一两步完成。
5. **演示效果强**：知识图谱、学习报告、情绪曲线和手势控制应具备直观的视觉表现。
6. **真实可用**：功能不只停留在概念展示，应尽量串成完整闭环。

## 11. 最终演示流程

项目演示可以按照以下流程进行：

1. 打开 LongMindStudy 首页，展示学习工作台。
2. 上传一份课程 PPT 或 PDF。
3. 系统解析资料，生成摘要、关键词和章节结构。
4. 打开知识图谱，检查 Neo4j 已连接，点击生成图谱。
5. 展示 Qwen 抽取出的课程概念关系网络，并说明结果已写入本地 Neo4j。
6. 打开思维导图，查看由资料独立生成的彩色分支脑图，并导出 Mermaid mindmap。
7. 搜索某个知识点，展开邻居节点，查看解释、例子和原文出处。
8. 选择两个知识点查找路径，展示 Neo4j 的关系查询能力。
9. 向 AI 提问课程相关问题。
10. 系统生成练习题，用户完成一组自测。
11. 展示答案解析、错题记录和薄弱知识点。
12. 打开学习报告，查看进度、正确率和复习建议。
13. 展示情绪状态提醒和音乐推荐。
14. 用手势完成翻页、确认答案或收藏内容。

## 12. 项目可行性分析

### 12.1 技术可行性

项目所需技术都有成熟工具支持。当前实现以 Electron 桌面端为主体，文档解析和 RAG 在本地模块中完成，AI 问答和知识图谱生成使用 Qwen / DashScope，图谱持久化和关系查询使用本地 Neo4j，情绪和手势交互使用摄像头与 MediaPipe 能力。

### 12.2 开发可行性

项目可以按模块逐步实现。优先完成资料上传、摘要、问答、知识图谱和测验功能，再加入情绪提醒、音乐推荐和手势控制。即使部分 AI 或识别能力以简化方式实现，也能形成完整可演示的软件闭环。

### 12.3 展示可行性

LongMindStudy 具有较强的课堂展示效果。资料解析、知识图谱、思维导图、AI 问答、自动测验、学习报告、音乐推荐和手势控制都能直观展示软件价值。思维导图可以作为知识图谱之外的第二种结构化展示方式：图谱负责关系查询，导图负责复习层级梳理，两者共同说明系统能把课程资料转化成可学习、可展示、可练习的内容。

## 13. 项目亮点

1. 以实际软件体验为核心，而不是只做概念或研究方案。
2. 将课程资料整理、AI 问答、自动练习和复习反馈整合成完整学习闭环。
3. 使用 Qwen + Neo4j 知识图谱把复杂课程内容变得结构化、可视化、可查询，并通过思维导图把资料整理为更适合复习的层级主题。
4. 通过错题本和学习报告帮助学生明确复习方向。
5. 通过情绪提醒和音乐推荐改善长时间学习体验。
6. 支持基础手势操作，让软件更自然、更有展示效果。
7. 目标是完成一个界面精美、功能实用、适合真实使用的桌面端原型。
