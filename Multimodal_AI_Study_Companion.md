# MindStudy：多模态智能学习伙伴系统项目方案

## 1. 项目名称

**中文名称：** MindStudy：多模态智能学习伙伴系统

**英文名称：** Multimodal AI Study Companion for University Courses

## 2. 项目背景

在大学课程学习过程中，学生通常需要面对大量 PPT、PDF、教材和课堂笔记等学习资料。传统学习方式存在以下问题：

- 课程资料数量多，学生难以快速抓住重点。
- 普通阅读器和笔记工具缺少智能化分析与个性化反馈。
- 长时间学习容易产生疲劳、焦虑、困惑和注意力下降。
- 学习交互方式主要依赖鼠标和键盘，缺乏自然、高效的人机交互体验。
- 学生很难及时了解自己的薄弱知识点和复习方向。

因此，本项目提出一个面向大学课程学习场景的多模态智能学习伙伴系统。系统融合 AI 学习辅助、情绪感知音乐交互和手势控制交互，帮助学生更高效地理解课程内容、管理学习过程并改善学习体验。

## 3. 项目目标

本项目旨在构建一个桌面端智能学习辅助平台，使学生能够上传课程资料，并获得自动化的知识整理、问答、测验、错题追踪和学习反馈。同时，系统通过摄像头识别学生的情绪状态和手势动作，实现更自然、更智能的多模态交互。

项目核心目标包括：

1. 支持上传 PPT、PDF 等课程资料。
2. 自动提取课程重点、关键词和知识结构。
3. 自动生成可视化知识图谱。
4. 支持基于课程内容的 AI 问答。
5. 自动生成练习题并追踪错题。
6. 记录学习进度并提供个性化复习建议。
7. 通过摄像头识别学习者情绪状态。
8. 根据情绪推荐背景音乐或休息建议。
9. 支持手势控制翻页、确认、截图、切题等操作。
10. 从人机交互角度研究多模态界面对学习效率和用户体验的影响。

## 4. 项目总体设计

本项目将三个方向整合为一个完整系统：

- **AI Study Companion：** 负责课程资料理解、知识图谱、问答、出题和学习追踪。
- **Emotion-aware Music Interaction：** 负责表情识别、情绪曲线、音乐推荐和学习状态调节。
- **Gesture-Control Interface：** 负责手势识别和自然交互控制。

系统不是单纯的聊天机器人，而是一个以学习任务为中心的智能交互平台。它将课程内容理解、学习行为分析、情绪感知和自然交互融合在同一个学习环境中。

## 5. 系统功能模块

### 5.1 课程资料上传模块 UploadModule

学生可以上传课程 PPT、PDF 或讲义文件。系统自动解析文件内容，提取文本、标题、章节结构、关键词和重要概念。

主要功能：

- PPT/PDF 文件上传
- 文本内容提取
- 章节结构识别
- 关键词提取
- 课程资料管理
- 学习资料预览

### 5.2 知识图谱模块 KnowledgeGraph

系统根据课程内容自动生成知识图谱或知识树，将课程知识点及其关系进行可视化展示。

示例结构：

```text
人机交互
├── 可用性
│   ├── 有效性
│   ├── 效率
│   └── 满意度
├── 用户体验
│   ├── 情感体验
│   └── 交互体验
└── 评估方法
    ├── SUS 量表
    ├── 用户访谈
    └── 可用性测试
```

主要功能：

- 自动生成知识树或知识网络
- 展示知识点之间的关联关系
- 点击节点查看解释、例子和相关原文
- 标记已掌握、未掌握和易错知识点
- 按章节、难度或掌握程度筛选知识点

HCI 研究价值：

- 研究知识图谱布局对学习效率的影响。
- 比较可视化知识树与传统文本摘要的学习效果。
- 帮助学生建立更清晰的课程知识结构。

### 5.3 AI 问答模块 QAEngine

学生可以围绕上传的课程资料进行提问。系统基于 RAG 检索增强生成技术，从课程资料中查找相关内容并生成回答。

示例问题：

- 这一章的核心概念是什么？
- Fitts 定律怎么理解？
- 帮我用简单例子解释认知负荷。
- 这页 PPT 可能会怎么考？
- 这个知识点和上一章有什么关系？

主要功能：

- 基于课程资料的智能问答
- 支持连续追问
- 回答时显示资料来源
- 支持简洁解释、详细解释、举例说明等回答风格
- 支持将回答保存为笔记

### 5.4 自动出题模块 QuizGenerator

系统根据课程重点自动生成练习题，帮助学生进行自测和复习。

支持题型：

- 单选题
- 多选题
- 判断题
- 简答题
- 概念匹配题

主要功能：

- 按章节生成题目
- 按知识点生成题目
- 按难度生成题目
- 自动批改客观题
- 为简答题提供参考答案
- 根据答题情况推荐复习内容

### 5.5 错题与进度追踪模块 ProgressTracker

系统记录学生的学习行为、答题结果和知识点掌握情况，形成可视化学习反馈。

主要功能：

- 学习时长统计
- 章节完成度统计
- 测验正确率统计
- 错题本管理
- 薄弱知识点识别
- 个性化复习建议
- 学习趋势可视化

HCI 研究价值：

- 帮助学生明确自己的薄弱环节。
- 减少无目标复习。
- 研究不同反馈方式对学习动力和学习效果的影响。

### 5.6 情绪感知模块 EmotionMonitor

系统通过摄像头识别学生学习过程中的表情状态，例如专注、开心、疲劳、焦虑、困惑等，并生成情绪变化曲线。

主要功能：

- 实时表情识别
- 学习状态检测
- 情绪变化曲线展示
- 疲劳检测
- 注意力下降提醒
- 学习状态记录

示例反馈：

> 系统检测到你可能已经有些疲劳，建议休息 3 分钟，或切换到轻量复习模式。

HCI 研究价值：

- 引入情绪计算。
- 探索情绪反馈对学习体验的影响。
- 实现自适应学习界面和个性化提醒。

### 5.7 情绪音乐推荐模块 MusicInteraction

系统根据学习者的情绪状态推荐合适的背景音乐，帮助学生维持专注、缓解疲劳或降低焦虑。

推荐规则示例：

| 学习状态 | 推荐音乐类型 | 系统策略 |
| --- | --- | --- |
| 专注 | 轻音乐、白噪音 | 保持低干扰播放 |
| 疲劳 | 舒缓但有节奏的音乐 | 提醒休息或切换学习模式 |
| 焦虑 | 放松音乐、自然声 | 降低刺激感 |
| 开心 | 轻快音乐 | 保持积极学习状态 |
| 困惑 | 降低音量或暂停 | 建议查看知识图谱或提问 |

主要功能：

- 情绪驱动音乐推荐
- 背景音乐播放
- 音量自动调节
- 用户反馈：有帮助 / 没帮助
- 根据用户反馈优化推荐策略

### 5.8 手势交互模块 GestureControl

系统通过摄像头识别学生手势，实现无接触式学习控制。

手势设计示例：

| 手势 | 功能 |
| --- | --- |
| 张手 | 下一页 / 展开知识点 |
| 握拳 | 暂停学习 / 暂停音乐 |
| 点赞 | 确认答案 |
| 比 V | 截图 / 收藏当前内容 |
| 左右挥手 | 上一题 / 下一题 |

主要功能：

- 实时手势识别
- PPT 或资料翻页控制
- 题目切换
- 答案确认
- 截图和收藏
- 音乐播放控制

HCI 研究价值：

- 探索自然用户界面。
- 降低鼠标和键盘依赖。
- 提升学习过程中的操作流畅性。
- 增强课堂演示效果。

## 7. 技术方案

### 7.1 前端技术

- React
- TypeScript
- Electron
- ECharts 或 D3.js
- Tailwind CSS 或 Ant Design

### 7.2 后端技术

- Python
- FastAPI
- LangChain 或 LlamaIndex
- Ollama / DeepSeek API
- FAISS / ChromaDB
- PyMuPDF
- python-pptx

### 7.3 情绪与手势识别

- OpenCV
- MediaPipe
- DeepFace
- 摄像头实时检测

### 7.4 音乐推荐

- Spotify API 或本地音乐库
- 基于规则的推荐算法
- 用户反馈优化机制

### 7.5 可选扩展

如果后期有相关设备，可以扩展：

- EEG 脑电信号
- EMG 肌电信号
- IMU 惯性传感器

这些扩展可以用于探索脑机接口或生理信号驱动的人机交互。

## 8. HCI 研究重点

本项目的重点不是简单实现 AI 聊天，而是围绕人机交互进行系统设计与实验研究。

可研究的问题包括：

1. 知识图谱界面是否比普通文本摘要更能提升学习效率？
2. 情绪反馈是否能降低学生学习疲劳感？
3. 手势交互是否能提升学习过程中的操作流畅性？
4. AI 提示方式如何影响学生的理解、记忆和学习信心？
5. 多模态反馈是否能提升学习参与感和满意度？
6. 不同界面布局对学习效率和认知负荷有什么影响？
7. 情绪音乐推荐是否能改善学习专注度和学习体验？

## 9. 实验设计

为了验证系统的人机交互效果，可以设计用户实验。

### 9.1 实验对象

招募大学生作为实验参与者，要求具有基本电脑操作能力，并正在学习或曾经学习相关课程内容。

### 9.2 实验分组

| 组别 | 使用工具 |
| --- | --- |
| 对照组 | 普通 PDF 阅读器、传统笔记工具 |
| 实验组 | MindStudy 多模态智能学习伙伴系统 |

### 9.3 实验流程

1. 为参与者提供相同的课程学习资料。
2. 对照组使用传统方式学习。
3. 实验组使用 MindStudy 系统学习。
4. 学习结束后完成相同的测验。
5. 收集学习时间、测验成绩和用户体验问卷。
6. 分析两组在学习效率、学习效果和用户满意度方面的差异。

### 9.4 评价指标

- 完成学习任务所需时间
- 测验正确率
- 知识点回忆率
- SUS 系统可用性评分
- 用户满意度
- 疲劳感自评
- 操作流畅度评分
- 情绪状态变化
- 手势识别准确率

## 10. 项目创新点

1. 将 AI 学习助手与多模态交互结合，提升学习系统的智能化程度。
2. 不只提供聊天问答，还提供知识图谱、自动出题、错题追踪和学习反馈。
3. 引入情绪识别，根据学习状态调整系统反馈方式。
4. 使用音乐推荐调节学习氛围，改善学习体验。
5. 支持手势控制，实现更自然的人机交互。
6. 具备明确的 HCI 研究价值，适合课程项目展示和报告撰写。
7. 可扩展脑电、肌电和 IMU 等设备，形成更完整的多模态交互平台。

## 11. 最终演示流程

项目演示可以按照以下流程进行：

1. 用户上传一份课程 PPT 或 PDF。
2. 系统自动解析资料内容。
3. 系统生成课程重点摘要和知识图谱。
4. 用户点击某个知识点查看解释和原文出处。
5. 用户向 AI 提问课程相关问题。
6. 系统根据课程内容自动生成测试题。
7. 用户答题后，系统自动判断并记录错题。
8. 系统展示学习进度、正确率和薄弱知识点。
9. 摄像头检测用户情绪状态，生成情绪变化曲线。
10. 如果检测到疲劳或焦虑，系统推荐休息或播放舒缓音乐。
11. 用户通过手势切换题目、确认答案或收藏知识点。
12. 最后展示学习报告和个性化复习建议。

## 12. 项目可行性分析

### 12.1 技术可行性

项目所需技术均有成熟工具支持。文档解析可以使用 PyMuPDF 和 python-pptx；AI 问答可以使用 RAG、Ollama 或 DeepSeek；情绪和手势识别可以使用 OpenCV、MediaPipe 和 DeepFace；桌面端界面可以使用 React、TypeScript 和 Electron 实现。

### 12.2 演示可行性

系统具有较强的展示效果。上传资料、生成知识图谱、AI 问答、自动出题、情绪曲线和手势控制都适合课堂展示。

### 12.3 报告可行性

项目包含明确的人机交互研究问题，可以围绕界面设计、交互方式、学习效率、情绪反馈和用户体验进行分析，因此便于撰写课程报告。

项目最终预期完成以下成果：

- 一个 Electron 桌面端原型系统。
- 支持课程资料上传和解析。
- 支持知识图谱可视化。
- 支持课程内容 AI 问答。
- 支持自动出题和错题追踪。
- 支持学习进度可视化。
- 支持情绪识别和情绪曲线展示。
- 支持情绪音乐推荐。
- 支持基础手势控制。
- 一份完整的人机交互课程项目报告。

## 14. 一句话总结

**MindStudy 是一个融合课程内容理解、知识可视化、AI 问答、自动测验、情绪感知、音乐调节与手势交互的多模态智能学习伙伴系统，旨在提升大学生的学习效率、专注度和交互体验。**
