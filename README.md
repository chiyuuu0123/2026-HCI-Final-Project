# LongMindStudy-龙龙多模态智能学习伙伴

LongMindStudy-龙龙多模态智能学习伙伴是一款面向大学生课程学习的桌面端智能学习软件。它的重点不是做理论研究，而是做出一个界面精美、流程顺畅、功能实用、由龙龙陪伴的学习助手，让学生能够把 PDF、Markdown 等课程资料变成可理解、可练习、可追踪的学习内容。

软件将 AI 课程问答、知识图谱、自动测验、错题追踪、学习进度、情绪状态提醒、背景音乐和手势控制整合在一个完整的学习工作台中，目标是让学习过程更清晰、更轻松、更有掌控感。

## 产品定位

LongMindStudy 面向需要高效复习课程资料的大学生，尤其适合期末复习、课堂资料整理、课程重点梳理和自测练习场景。

它希望解决这些实际问题：

- 资料太多，不知道从哪里开始看。
- PDF 和 Markdown 内容零散，难以形成完整知识结构。
- 学习时遇到概念不懂，需要快速追问。
- 复习缺少练习题和错题反馈。
- 学习时间久了容易疲劳、分心或焦虑。
- 操作频繁依赖鼠标键盘，不够自然和沉浸。

## 核心功能

### 课程资料库与文件解析

用户可以自行创建和切换课程。每门课程都有一套独立资料库、课程说明、当前阅读文件、学习批注和编辑内容，不同课程之间互不混杂。

在一个课程内，用户可以一次导入多个 PDF 或 Markdown 文件。资料阅读页左侧会显示当前课程的资料库，用户可以在多个文件之间切换阅读。桌面端会保存资料的本机路径，关闭软件后再次打开时，不需要重新上传 PDF 或其他资料，只要原文件还在原路径，就可以继续读取。

当前实现：

- 课程资料库：支持新建课程、切换课程、编辑课程名称和课程说明，每门课程独立保存资料列表和学习信息。
- 多文件导入：支持在同一门课程内导入多个 PDF、MD 文件，并在左侧资料库中切换阅读。
- PDF：支持导入、内嵌阅读、自动提取文字、学习批注、画笔批注、插入图片、添加空白页、删除指定页、保存 Markdown 笔记、导出 PDF 副本。
- Markdown：支持导入、预览、编辑、保存原文件或另存为副本。
- AI 阅读窗口：资料阅读页右侧提供 AI 阅读辅助，可以解析当前 Markdown 正文或 PDF 滚动停留页自动提取出的文字，并把英文内容翻译成中文。

### 智能摘要、知识图谱与思维导图

系统根据课程内容生成章节摘要、Qwen + Neo4j 交互式知识图谱和独立思维导图。知识图谱用于查看概念关系、搜索节点、展开邻居和查找路径；思维导图直接从课程资料整理层级主题，用彩色分支梳理复习结构，支持聚焦主题、展开收起分支，并导出 Mermaid mindmap 文本用于报告展示。

### 课程 AI 问答

用户可以围绕上传资料提问，例如“这一章重点是什么”“这个概念怎么理解”“这段资料可能怎么考”。系统基于课程资料回答，并尽量给出来源和解释。

### 自动测验与错题本

系统可以根据课程资料、章节、知识点、难度和历史答题情况生成自适应练习，支持选择题、判断题、简答题和概念匹配题。答题后会记录错题、错因、来源片段和下次复习时间，并推荐需要补强的知识点。

### 学习进度面板

软件提供学习时长、章节完成度、测验正确率、薄弱知识点和复习建议，让学生知道自己学到哪里、哪里还不熟。

### 情绪状态提醒

系统可以调用本机摄像头，展示实时预览，并根据画面亮度给出基础学习状态反馈。当前版本已实装摄像头开启、停止、权限请求、顶部状态同步和状态页实时预览。

### 背景音乐推荐

软件根据学习状态推荐低干扰背景音乐、白噪音、自然声或舒缓音乐，帮助用户维持专注或缓解疲劳。

### 手势控制

软件支持用摄像头识别基础手势，实现翻页、暂停、确认答案、截图收藏和切换题目等无接触操作。

| 手势     | 功能                |
| -------- | ------------------- |
| 张手     | 下一页 / 展开知识点 |
| 握拳     | 暂停学习 / 暂停音乐 |
| 点赞     | 确认答案            |
| 比 V     | 截图 / 收藏当前内容 |
| 左右挥手 | 上一题 / 下一题     |

## 典型使用流程

1. 打开 LongMindStudy，进入学习工作台。
2. 创建或切换到当前要复习的课程。
3. 在当前课程内上传多个 PDF 或 Markdown 资料。
4. 在资料阅读页左侧资料库中切换文件，阅读 PDF 或编辑 Markdown。
5. 系统解析资料并生成摘要、关键词、知识图谱和思维导图。
6. 用户浏览知识图谱查看概念关系，或浏览独立思维导图查看复习主题。
7. 用户向 AI 提问课程相关问题，或让系统生成练习题进行自测。
8. 软件记录错题、正确率、薄弱知识点和当前课程学习批注。
9. 学习过程中检测到疲劳或困惑时，系统给出提醒，也可以播放推荐音乐或用手势切换页面。
10. 最后查看学习报告和个性化复习建议。

## 软件界面规划

LongMindStudy 的界面目标是清爽、现代、易用，适合作为一个真正可演示的软件原型。

主要页面包括：

- **首页 / 工作台**：展示最近资料、今日学习进度、待复习内容和快速入口。
- **资料阅读页**：左侧当前课程资料库，中间资料预览，右侧 AI 助手、批注和编辑区。
- **首页大日历**：在工作台记录考试、DDL、答辩、面试等重要事件。
- **TodoList 页**：集中管理学习日程表和今日待办，支持进度、状态和重要程度。
- **知识图谱页**：可视化展示课程知识结构，支持节点筛选、状态标记、搜索定位、邻居展开、路径高亮和 Mermaid 导出。
- **思维导图页**：从课程资料独立生成彩色分支脑图，支持主题聚焦、展开收起和 Mermaid mindmap 导出。
- **AI 问答页**：支持连续追问、来源展示和保存为笔记。
- **测验页**：支持自适应练习、快速诊断、薄弱巩固、今日复习和考前冲刺，展示题目、答案、解析、来源片段和错题记录。
- **学习报告页**：展示学习时长、正确率、薄弱知识点和复习建议。
- **音乐与状态栏**：展示当前学习状态、音乐推荐和摄像头识别状态。
- **龙龙桌宠助手**：常驻右下角提醒日程、同步状态与音乐建议，并预留资料库 RAG 问答接口；桌面端最小化后仍保留奶龙悬浮窗，点击可唤醒软件。

## 本地运行

前置要求：

- 安装 Node.js 和 npm。
- 安装 Neo4j Desktop。项目脚本会优先连接 Neo4j Desktop 中已经启动的 DBMS；如果没有运行中的 DBMS，会自动使用 Neo4j Desktop 自带的 Neo4j/Java 运行时启动一个 LongMindStudy 专用本地库。
- 本地端口 `7474` 和 `7687` 没有被其他 Neo4j 实例占用。

安装依赖：

```bash
npm install
```

启动桌面端应用：

```bash
npm start
```

`npm start` 只启动 LongMindStudy。正式演示前建议先运行 `npm run neo4j:start`，确保 Neo4j Desktop 里的本地数据库已经启动并能通过 Bolt 连接。

安装完成后会自动准备前端本地图标资源。如果需要手动重新生成，可以运行：

```bash
npm run prepare:assets
```

龙龙桌宠已内置 Hugging Face `NaiLong-Voice-Clone` 精选参考音频。如果换机后素材缺失，可以重新准备：

```bash
npm.cmd run prepare:voice
```

启动桌面端应用：

```bash
npm start
```

Windows PowerShell 如果提示 `npm.ps1` 无法运行，可以改用：

```bash
npm.cmd install
npm.cmd start
```

只启动 Electron 的调试命令：

```bash
npm run start:app
```

这个命令等价于 `npm start`，主要用于保留调试入口。

如果 Electron 运行时下载较慢，可以先设置镜像源后再安装：

```bash
$env:ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"
npm.cmd install
```

如果只是查看前端原型，也可以直接用本地静态服务打开 `frontend/index.html`。但正式演示建议使用 Electron 窗口启动，因为“导入资料”等能力会调用桌面端文件选择框。

## Neo4j 本地管理

本项目采用 Neo4j Desktop 方案管理本地图数据库。脚本会尝试打开 Neo4j Desktop 并检查 Bolt 连接；如果 Neo4j Desktop 中没有正在运行的 DBMS，脚本会使用 Neo4j Desktop 已下载到本机的 Neo4j/Java 运行时，在 `%LOCALAPPDATA%\LongMindStudyNeo4j` 下启动一个 LongMindStudy 专用本地库。

首次配置：

1. 安装 Neo4j Desktop。
2. 运行 `npm run neo4j:start`，脚本会创建 `.local/neo4j.env` 并打开 Neo4j Desktop。
3. 如果 Neo4j Desktop 已有运行中的 DBMS，脚本会直接连接它。
4. 如果没有运行中的 DBMS，脚本会自动解压 Neo4j Desktop 自带发行包到 `%LOCALAPPDATA%\LongMindStudyNeo4j`，设置初始密码，并启动本地库。
5. 运行 `npm run neo4j:check`，看到连接成功后再演示图谱功能。

启动 Neo4j：

```bash
npm run neo4j:start
```

检查连接：

```bash
npm run neo4j:check
```

关闭 Neo4j Desktop 进程：

```bash
npm run neo4j:stop
```

默认连接信息：

```text
Browser: http://localhost:7474
Bolt: bolt://localhost:7687
Username: neo4j
Password: 保存在 .local/neo4j.env
Runtime: %LOCALAPPDATA%\LongMindStudyNeo4j
```

`.local/` 已加入 `.gitignore`。Neo4j 密码只保存在本机，不提交到仓库。`%LOCALAPPDATA%\LongMindStudyNeo4j` 是本机运行时和数据库目录，也不通过 Git 同步。团队成员拉取代码后各自运行 `npm install`、安装 Neo4j Desktop，再运行 `npm run neo4j:start` 和 `npm run neo4j:check` 即可。

如果需要覆盖默认配置，可以在启动前设置环境变量：

```bash
$env:NEO4J_URI="bolt://localhost:7687"
$env:NEO4J_USERNAME="neo4j"
$env:NEO4J_PASSWORD="your_local_password"
npm.cmd start
```

## Qwen API Key 配置

知识图谱生成、资料问答和摘要使用 Qwen / DashScope。Key 不要写进源码。

推荐方式：

1. 打开 LongMindStudy。
2. 点击界面中的 `Key` 按钮。
3. 输入 DashScope / Qwen API Key 并保存。

也可以用环境变量：

```bash
$env:DASHSCOPE_API_KEY="your_api_key_here"
npm.cmd start
```

没有 Qwen API Key 时，知识图谱仍会使用本地规则生成，并尽量写入 Neo4j，保证课堂演示不中断。

## 音乐推荐 API Key 配置

音乐推荐已经和通用 AI 问答拆开。课程问答、知识图谱和摘要仍使用 `DASHSCOPE_API_KEY` / `QWEN_API_KEY`，但“状态与音乐”页面在歌曲播放结束后让 AI 选择下一首时，只读取独立环境变量：

```text
Music_Recommand_Key
```

临时设置当前 PowerShell 会话：

```powershell
$env:Music_Recommand_Key="your_music_recommend_api_key_here"
npm.cmd start
```

持久写入 Windows 用户环境变量：

```powershell
[Environment]::SetEnvironmentVariable("Music_Recommand_Key", "your_music_recommend_api_key_here", "User")
```

持久写入后需要关闭当前 PowerShell，重新打开终端并重启 Electron 应用。

如果没有配置 `Music_Recommand_Key`，音乐播放器仍可正常播放；AI 选下一首会失败并自动降级为本地随机选择。

## 知识图谱使用方法

1. 在 Neo4j Desktop 中启动本地 DBMS，并运行 `npm run neo4j:check` 确认连接成功。
2. 运行 `npm start` 打开 LongMindStudy，确保知识图谱页的 Neo4j 状态显示为已连接。
3. 在首页点击 `导入资料`，选择 PDF 或 Markdown。
4. 进入 `知识图谱` 页面，点击 `生成知识图谱`。
5. 系统会优先使用 Qwen 抽取概念和关系，写入 Neo4j，再渲染为可交互关系网络。
6. 在搜索框输入知识点或关键词，点击 `搜索定位`。
7. 点击节点后可查看摘要、章节、难度、掌握度、来源片段和相关节点。
8. 点击 `展开邻居` 可以从 Neo4j 查询当前节点的一度关系。
9. 选择路径起点和终点，点击 `查找路径`，图谱会高亮最短关系路径。
10. 点击 `导出 Mermaid` 可以复制 `graph TD` 文本到报告或演示材料中。
11. 点击 `练习该知识点` 可以筛选已有题组并进入自动测验。

## 思维导图使用方法

1. 先导入资料并生成学习内容，系统会从资料标题、章节标题和关键词独立生成思维导图。
2. 进入 `思维导图` 页面，选择 `彩色分支` 或 `紧凑层级`。
3. 点击分支标题可以展开或收起该分支。
4. 点击导图节点可以在右侧查看摘要、来源和复习提示。
5. 在聚焦下拉框选择一个主题后点击 `聚焦`，只查看该主题及其子分支。
6. 点击 `导出导图文本` 可以查看当前独立导图的 Mermaid mindmap 文本。
7. 点击 `导出 Mindmap` 可以复制 Mermaid `mindmap` 文本，用于报告或课堂展示。

## 自动测验使用方法

1. 先导入课程资料并生成知识图谱，系统会同时准备题组和来源片段。
2. 进入 `自动测验` 页面，在练习模式中选择：
   - `自适应练习`：优先安排低掌握度、刚答错和尚未练习的资料题目。
   - `快速诊断`：每个核心知识点抽一题，快速找薄弱点。
   - `薄弱巩固`：集中练习错题和掌握度低的节点。
   - `今日复习`：按错题复习队列重做到期题目。
   - `考前冲刺`：混合难度并优先提高较难题比例。
3. 答题后查看解析、正确答案、来源片段和下一步操作。
4. 简答题会优先使用 Qwen 进行语义评分，返回分数、缺失要点和具体反馈；没有可用 API Key 时自动使用关键词规则评分。
5. 右侧 `今日复习` 会显示待复习错题，答对两轮后自动标记为已复习。

## 常见问题

- Neo4j Desktop 未安装：运行 `winget install -e --id Neo4j.Neo4jDesktop`，或从官网安装。
- Neo4j 未连接：运行 `npm run neo4j:start`。如果仍失败，确认 Neo4j Desktop 已完成首次初始化，或检查端口是否被占用。
- 端口被占用：检查本机是否已有 Neo4j 占用 `7474` 或 `7687`，关闭旧实例后重试。
- Neo4j 密码错误：让 `.local/neo4j.env` 中的 `NEO4J_PASSWORD` 与 Neo4j Desktop DBMS 密码保持一致。
- Qwen key 缺失或未开通额度：图谱会降级为本地规则生成，简答题会降级为关键词评分；需要真实 AI 抽取和语义评分时，在界面 Key 设置中保存可用的 DashScope API Key。
- 想清空演示数据：停止 Neo4j 后删除 `%LOCALAPPDATA%\LongMindStudyNeo4j`，或在 Neo4j Desktop 中清空/重建 DBMS，或在知识图谱页重新生成当前课程图谱。

固定音频位于 `frontend/assets/longlong-voice`，主要包括：

- `boot-01.wav` 至 `boot-06.wav`：上线问候。
- `poke-01.wav` 至 `poke-08.wav`：摸摸反馈。
- `tip-01.wav` 至 `tip-10.wav`：学习提醒。
- `ai-thinking.wav`、`ai-answer.wav`：AI 思考提示和回答开场。

如果之后要新增固定文案，可以临时使用 GPT-SoVITS / CosyVoice 等本地服务生成音频。生成完成后，把音频放进 `frontend/assets/longlong-voice`，再在 `frontend/companion.html` 的 `hardcodedLonglongAudio` 里添加对应关系。

本地生成服务可以在启动前设置：

```bash
$env:LONGLONG_TTS_ENDPOINT="http://127.0.0.1:9880/tts"
$env:LONGLONG_TTS_REFERENCE_FILE="reference.wav"
$env:LONGLONG_TTS_PROMPT_TEXT="啊，我才不要这样，好害羞啊。"
npm.cmd start
```

默认请求格式按 GPT-SoVITS 的 `/tts` JSON 接口组织，包含文本、参考音频路径、参考文本和语言。这个接口现在主要用于制作新的固定音频，不作为桌宠日常运行依赖。

## 已实现的文件能力

当前版本已经实装了基础课程资料库和文件工作流：

1. 在首页顶部可以新建课程、切换课程、编辑课程名称和课程说明。
2. 点击 `导入资料` 选择本地课程文件，支持一次导入多个 PDF 或 MD 文件。
3. 每门课程拥有独立资料库，导入后的文件会显示在资料阅读页左侧，可以随时切换阅读。
4. 资料条目可以从当前课程资料库移除；移除只影响软件内资料库，不会删除本地原文件。
5. 桌面端会保存资料的本机路径，关闭软件后重新打开时，不需要重新上传 PDF；只要原文件没有移动或删除，软件会按保存路径继续读取。
6. 资料阅读页提供 `全屏阅读` 按钮，可以隐藏普通工作台干扰，进入沉浸式三栏阅读状态。
7. 导入 PDF 后，可以在资料阅读页查看 PDF，并在右侧填写批注；同时支持画笔批注、插入 PNG/JPG 图片、在当前页后添加空白页、删除当前页，并导出新的 PDF 副本。
8. 导入 Markdown 后，可以直接预览和编辑，支持保存回原文件或另存为副本。
9. 资料阅读页右侧提供 AI 阅读窗口：Markdown 会自动带入正文；PDF 滚动到不同页面时会自动切换当前页文字，然后进行内容解析或英文转中文。
10. 首页工作台提供大日历，用于记录考试、DDL、答辩、面试等重要事件；独立 TodoList 页提供类似飞书多维表格的学习计划表和今日待办，支持筛选状态、调整进度和设置重要程度。
11. 状态与音乐页可以开启本机摄像头，显示实时画面，并同步顶部摄像头状态。
12. 按住 `Ctrl` 滑动鼠标滚轮可以调整全局页面缩放比例；`Ctrl + 0` 可以恢复到 100%。
13. 右下角常驻龙龙桌宠助手，可以拖拽调整位置，提醒学习日程、提示状态和音乐建议，并提供资料库 RAG 问答接口占位；问候和提醒语音使用内置固定音频包，不再运行时合成或回退系统朗读。
14. 知识图谱页已接入 Qwen + Neo4j：支持本地数据库连接检查、生成图谱、从 Neo4j 加载、搜索节点、展开邻居、路径高亮、原文片段查看和 Mermaid 导出。

说明：当前版本只允许导入 PDF 和 Markdown；PDF 的插图、加页和删页会作用在当前副本上，不会直接破坏原文件；Markdown 只有在用户主动点击保存原文件时才会写回本地文件。
