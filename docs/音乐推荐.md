# 音乐推荐说明

## 页面功能

“状态与音乐”页面下方是音乐播放器和歌单系统。播放器支持：

- 播放、暂停、上一首、下一首。
- 拖动进度条跳转播放位置。
- 调整音量。
- 点击歌单中的曲目切换播放。

## 情绪驱动歌单

系统会先根据当前稳定状态选择基础歌单：

| 状态 | 基础歌单倾向 |
| --- | --- |
| 专注 | 白噪音、轻钢琴、低干扰曲目 |
| 疲劳 | 慢速、舒缓、适合休息的曲目 |
| 疑惑 | 低干扰、适合整理思路的曲目 |
| 焦虑 | 低音量、舒缓、降低紧张感的曲目 |
| 走神 | 轻节奏、帮助拉回注意力的曲目 |
| 轻松 | 平稳、自然、保持节奏的曲目 |
| 离开 | 暂停推荐，不主动添加歌曲 |

## AI 加歌逻辑

当稳定状态发生变化时，前端会向已有的 `window.mindStudy.ai.askQuestion` 发送一次音乐推荐请求。请求内容包含：

- 当前状态和状态细节。
- 当前歌单中的曲目。
- 可添加歌曲列表。

AI 只能从“可添加歌曲列表”中选择，不能编造歌曲或文件路径。

期望返回 JSON：

```json
{"add":true,"title":"歌曲标题","reason":"一句话原因"}
```

或：

```json
{"add":false,"reason":"一句话原因"}
```

当前实现会解析 `answer` 字段中的 JSON。如果 `add=true` 且 `title` 命中候选歌曲，则把这首歌加入当前状态歌单；如果未命中候选歌曲或 `add=false`，则不修改歌单。

## 防抖策略

为了避免摄像头识别循环频繁调用 AI：

- `away` 状态不触发 AI 加歌。
- 同一个状态 30 秒内最多请求一次。
- 上一次 AI 请求未完成时，不发起新的请求。

## 候选歌曲

候选歌曲来自 `frontend/app.js` 中的 `aiMusicCandidateTracks`，所有路径都指向 `frontend/assets/music` 下已有的 mp3 文件。添加新候选歌时，需要保证文件存在，并使用：

```js
{ title: "歌名", artist: "歌手名", src: "./assets/music/歌名-歌手名.mp3", moods: ["focused"] }
```
