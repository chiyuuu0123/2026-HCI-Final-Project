# 音乐推荐

## 规则

播放器使用固定歌单，不再让 AI 动态加歌。

播放中情绪变化时，只更新当前情绪状态，不打断当前歌曲。当前歌曲自然播放结束后，再让 AI 根据当前情绪从固定歌单里选择下一首。

流程：

```text
歌曲播放结束
-> 读取当前情绪
-> 把固定歌单、当前情绪、最近播放记录发给 AI
-> AI 返回下一首歌曲的 index
-> 播放该歌曲
```

## AI 返回格式

AI 只需要返回 JSON：

```json
{"index":3,"reason":"适合当前状态"}
```

前端只使用 `index`，`reason` 只作为说明。

## 兜底

如果 AI 不可用、请求失败或返回格式错误，前端会本地随机选择下一首：

1. 优先选择 `moods` 包含当前情绪的歌曲。
2. 尽量避开最近播放过的歌曲。
3. 没有匹配时从固定歌单随机选择。

## 配置

### 播放器配置

播放器功能参数在：

```text
frontend/.env
```

当前相关参数：

```env
MUSIC_AI_NEXT_TRACK_ENABLED=true
MUSIC_AI_NEXT_TRACK_RECENT_LIMIT=3
```

| 参数 | 作用 |
| --- | --- |
| `MUSIC_AI_NEXT_TRACK_ENABLED` | 是否在歌曲结束时让 AI 选择下一首。 |
| `MUSIC_AI_NEXT_TRACK_RECENT_LIMIT` | 选择下一首时参考的最近播放记录数量。 |

修改 `.env` 后需要重启 Electron 应用。

### API Key 配置

音乐推荐不使用通用的 `DASHSCOPE_API_KEY` / `QWEN_API_KEY`，而是单独读取：

```text
Music_Recommand_Key
```

临时设置当前 PowerShell 会话：

```powershell
$env:Music_Recommand_Key="你的音乐推荐 API Key"
npm.cmd start
```

持久写入 Windows 用户环境变量：

```powershell
[Environment]::SetEnvironmentVariable("Music_Recommand_Key", "你的音乐推荐 API Key", "User")
```

持久写入后需要重新打开 PowerShell 并重启 Electron 应用。

如果没有配置该 Key，音乐播放不受影响；AI 选下一首会自动降级为本地随机选择。
